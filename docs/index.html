EEG Card Game
Decision Intelligence

I designed and implemented an end-to-end EEG machine learning workflow that turns raw multi-player brain-signal sessions into role prediction models. The system is built for reproducible research and readable engineering.

Project Idea

During a 3-player card game, each event contains EEG from all players. The challenge is to classify each player's role: played, next/current, or observer.

This mirrors real-world applied ML: noisy sensors, event context, class imbalance, and strict leakage prevention.

Motivational Example

Imagine a recruiter asks: “Can you take raw scientific data and ship a model pipeline?” This project is my answer: from MATLAB extraction to model comparison and reporting, with clear checkpoints and diagnostics.

Outcome: A reusable template for biosignal product prototyping and ML experimentation.

How I Built It

PhaseWhat I implemented
1. IngestionLoaded EEG tensors and labels from MAT files with shape checks.
2. LabelingMapped each player-event pair to role classes (played/next/observer).
3. Dataset splitBuilt event-wise train/test split to reduce leakage risk.
4. TrainingCompared CNN baseline vs patch-based Transformer models.
5. EvaluationTracked Accuracy, Balanced Accuracy, Macro-F1, confusion matrix.
