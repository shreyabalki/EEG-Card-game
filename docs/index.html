The Challenge

Can a neural network read cognitive intent from passive EEG?

🧠

The Research Question

During a card game, each player is in a different cognitive state depending on their role: the one who just played, the one whose turn is next, or the one passively observing. These subtle differences produce detectable EEG signatures — but can a model learn them?

🎯

Why It Matters

Passive cognitive state decoding is a core challenge in BCI research. Unlike explicit motor imagery tasks, players here have no BCI intent — the signal is ambient and naturalistic. Success generalizes to adaptive interfaces, cognitive load monitoring, and neuroergonomics.

Classification Target: Three Cognitive Roles

Label 0
Played
Player who just took a turn — post-decision state, reduced active attention
Label 1
Current / Next
Player whose turn is active or imminent — heightened anticipation and engagement
Label 2
Observer
Passive player — monitoring but not acting, lower cognitive load

Architecture

Two Architectures, One Unified Trainer

Architecture 1

Conv1D CNN

Input (B, 32ch, 801t)
↓
Conv1d 32→64, k=7 · BN · ELU · MaxPool
↓
Conv1d 64→128, k=5 · BN · ELU · MaxPool
↓
Conv1d 128→256, k=3 · BN · ELU · AdaptivePool
↓
Linear(256 → n_classes)

Local temporal feature extraction. Fast to train. Strong inductive bias for sequential signals.

Architecture 2

Patch Transformer

Input (B, 32ch, 801t)
↓
Patch Embedding: ~16 patches × (32 × 50)
↓
Linear Projection → d_model=128
↓
TransformerEncoder: 4 heads, 2 layers, ff=256
↓
Mean Pool over patches
↓
Linear(128 → n_classes)

Global temporal attention via patches. Captures long-range EEG dependencies efficiently.

Key Engineering Decision: Event-Wise Splits

Random sample-level splits would allow EEG from the same card game event to appear in both train and test sets — creating temporal leakage. Instead, entire events are grouped and held out, ensuring the model genuinely generalizes to unseen game moments.

❌ Random Split (Leaky)
E1-P1E1-P2E1-P3 E2-P1E2-P2E2-P3
Same event in both splits
✓ Event-Wise Split (Clean)
E1-P1E1-P2E1-P3 E2-P1E2-P2E2-P3
Entire events held out together

What This Project Proves

  • Built end-to-end EEG classification pipeline in PyTorch — MATLAB ingestion through CNN/Transformer training — for passive player role detection using only raw brainwave signals.
  • Engineered event-wise stratified train/test splits across 22 EEG sessions to eliminate temporal data leakage and ensure unbiased model evaluation.

  • Authored 14 modular Python scripts covering data QC, dataset engineering, model training, and sequence analysis — all parameterized via argparse CLI.
